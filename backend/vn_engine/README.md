# Visual Novel Engine

The Visual Novel Engine is a Python-based engine built with Pygame designed to play interactive stories generated by the AI storytelling system. It reads a structured `screenplay.json` file and renders the story with characters, backgrounds, music, and choices.

## Overview

The engine serves as the playback mechanism for the stories created by the backend services. It is designed to be modular and data-driven, meaning the entire experience is defined by the JSON data it loads. It handles:
-   Scene transitions
-   Dialogue display
-   Character sprite management (expressions/poses)
-   Background changes
-   Music playback
-   Branching narrative choices

## Architecture

The engine follows a modular architecture:

*   **`core.py`**: Contains the `GameEngine` class, which is the main entry point. It manages the game loop, event handling (input), and initializes other subsystems.
*   **`loader.py`**: The `StoryLoader` class is responsible for loading the `screenplay.json` file and parsing it into a usable structure. It provides helper methods to retrieve assets and scene data.
*   **`state_manager.py`**: The `SceneManager` and `GameState` classes track the current state of the game (current chapter, scene, dialogue line, active background, etc.). It implements the logic for advancing the story (`next()`) and handling choices.
*   **`stage.py`**: The `StageDirector` coordinates the visual representation of the current state. It updates the renderer based on the `GameState`.
*   **`graphics.py`**: Handles low-level Pygame rendering, sprite management, and window display.
*   **`converter.py`**: A utility script that aggregates the fragmented AI generation outputs (story outline, individual chapters, and scenes) into a single, unified `screenplay.json` file optimized for the engine.
*   **`main.py`**: The CLI entry point to run the engine.

## Data Flow

1.  **Generation**: The AI services generate a story outline, chapter lists, and detailed scene files (JSON). Assets (images, audio) are generated and saved to the output directory.
2.  **Conversion**: The `converter.py` script scans the output directory for a specific story ID. It combines the `story_outline.json`, chapter lists, and detailed scene files into one `screenplay.json`. It also maps asset filenames to relative paths.
3.  **Loading**: The `GameEngine` initializes a `StoryLoader` with the path to `screenplay.json`.
4.  **Execution**: The `SceneManager` iterates through the data. When `next()` is called (e.g., on mouse click), it advances the internal pointers and returns a "frame" object containing the current text, background, and character states.
5.  **Rendering**: The `StageDirector` takes this frame and updates the `Render` system to draw the screen.

## Screenplay Format

The `screenplay.json` has the following structure:

```json
{
  "metadata": {
    "title": "Story Title",
    "logline": "...",
    "main_characters": [...]
  },
  "assets": {
    "characters": { "char_name": "path/to/image.png" },
    "backgrounds": { "bg_name": "path/to/image.png" },
    "audio": { ... }
  },
  "story": {
    "chapters": [
      {
        "id": "chapter_1",
        "title": "Chapter 1",
        "scenes": [
          {
            "id": "scene_1",
            "content": {
              "main_dialogue": [
                {
                  "speaker": "Hero",
                  "text": "Hello world!",
                  "character_pose_expression": "happy"
                }
              ],
              "choices_and_branches": [...]
            }
          }
        ]
      }
    ]
  }
}
```

## Installation & Usage

### Dependencies

The engine requires `pygame`.

```bash
pip install pygame
```

### Running the Engine

To run a generated story:

1.  **Convert the story** (if not already done):
    ```bash
    python backend/vn_engine/converter.py <story_id>
    ```
    *   `<story_id>` is the folder name in `backend/services/output/` (e.g., `2`).

2.  **Run the engine**:
    ```bash
    python backend/vn_engine/main.py --screenplay backend/services/output/<story_id>/screenplay.json
    ```

    If you run it without arguments, it will try to find a default screenplay in the output directory.

    ```bash
    python backend/vn_engine/main.py
    ```

### Controls

*   **Left Click / Space / Enter**: Advance dialogue.
*   **Mouse Click on Choice**: Select an option during branching points.
*   **Esc**: Quit the engine.

## Key Components

*   **`converter.py`**:
    *   `StoryConverter`: Class that builds the `screenplay.json`.
    *   Methods: `scan_assets`, `build_metadata`, `build_story`.

*   **`loader.py`**:
    *   `StoryLoader`: Provides read-only access to story data.
    *   Methods: `get_scene_content`, `get_asset_path`.

*   **`state_manager.py`**:
    *   `GameState`: Dataclass holding mutable state (pointers, flags).
    *   `SceneManager`: Logic core. Handles `next()` transitions, choices, and jumps between scenes.
